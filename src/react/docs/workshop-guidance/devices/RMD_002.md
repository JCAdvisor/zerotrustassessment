# 002: Dispositivos groups

**Implementation Effort:** Low – IT and Security Operations teams need to create new Dispositivos groups.

**User Impact:** Low - The end users will not experience any new prompt or changes on their Dispositivos.

## Overview
1. **Overview of Intune Grouping and Targeting Concepts**:
   - **Azure Active Directory (Azure AD) Groups**: Intune primarily uses Azure AD groups for grouping and targeting. These groups allow you to assign apps, policies, and other workloads to users and Dispositivos. They're also used for role-based administration (RBAC) as scope groups.
   - **Virtual Groups**: The "All users" and "All Dispositivos" assignments are Intune's virtual groups. They exist by default in all Intune tenants and don't require manual management. They're highly scalable and optimized.
   - **Filters**: Filters allow you to narrow the assignment scope of apps and policies. You can target user or Dispositivos groups and filter Dispositivos based on specific properties. Filtering is high-performance and evaluated during Dispositivos check-in without pre-computation¹.

2. **Benefits of Dispositivos Groups**:
   - **Granularity**: Dispositivos groups allow precise targeting. You can create groups based on Dispositivos properties (e.g., OS version, compliance status).
   - **Efficiency**: Assigning policies to smaller Dispositivos groups reduces synchronization overhead and speeds up deployments.
   - **Troubleshooting**: Dispositivos groups help isolate issues by narrowing down the affected Dispositivos.

3. **Drawbacks and Considerations**:
   - **Synchronization Backlogs**: Large Azure AD groups with many users or Dispositivos can cause synchronization delays. Policy and app deployments take longer to reach managed Dispositivos¹.
   - **Tattooed Settings**: If settings or apps are applied before Dispositivos filters are evaluated, troubleshooting unexpected configurations becomes challenging².

4. **Dispositivos Groups vs. User Groups**:
   - **Dispositivos Groups**: Ideal for Dispositivos-specific policies (e.g., security settings, app installations).
   - **User Groups**: Useful for user-centric policies (e.g., user-targeted apps, conditional access).
