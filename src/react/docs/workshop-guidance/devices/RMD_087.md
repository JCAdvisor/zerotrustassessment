# 087: Compliance policy

## Overview

1. **Compliance Settings**:
   - **Microsoft Defender for Endpoint**: You can set a maximum allowed machine risk score for Dispositivos evaluated by Microsoft Defender for Endpoint. Dispositivos exceeding this score are marked as noncompliant.
   - **Dispositivos Health**: Specify the maximum allowed Dispositivos threat level evaluated by your mobile threat defense service. Dispositivos exceeding this threat level are marked noncompliant.
   - **System Security**: Configure settings related to password rules, minimum or maximum OS versions, app restrictions, and more.

2. **Benefits**:
   - **Security**: Compliance policies help protect organizational resources by ensuring Dispositivos meet security standards.
   - **Granularity**: You can target policies at groups of Dispositivos, not users, allowing precise control.
   - **Conditional Access**: Compliant Dispositivos can access enterprise resources protected by Conditional Access policies.

3. **Drawbacks**:
   - **Complexity**: Setting up and managing policies requires understanding of Android Enterprise and Intune.
   - **User Impact**: Noncompliant Dispositivos may lose access to resources, affecting productivity.
