# 011: Offboarding Strategy

## Overview
When offboarding users from **Microsoft Intune**, consider the following best practices based on your specific scenario:

1. **Deleting a User**:
   - If you're completely removing a user from **Azure AD**, follow these steps:
     - **Retire or remote wipe Dispositivos**: Ensure all Dispositivos associated with the user are retired or wiped before deleting them from Azure AD. This prevents issues with Dispositivos management.
     - **Remove from Intune Administrator groups**: Take the user out of any Azure AD security groups assigned Intune Administrator roles.
     - **Delete the user from Azure AD**: Once deleted, Microsoft Endpoint Manager will automatically remove the user from Intune reports, Dispositivos enrollment manager (DEM) accounts, and other configurations.

2. **Keeping a User**:
   - If you want to preserve the user's account in Azure AD but prevent them from enrolling Dispositivos:
     - **Retire or remote wipe Dispositivos**: Clean up Intune reports by retiring or wiping Dispositivos enrolled by the user.
     - **Remove from Intune Administrator groups**: Similar to the deletion process, remove the user from any relevant security groups.
     - **Restrict Dispositivos enrollment**: Add the user to an Azure AD security group with a Dispositivos type enrollment restriction blocking all platforms.
     - **Revoke Android Enterprise tokens**: If applicable, revoke any Android Enterprise tokens granted to prevent new Dispositivos enrollments.
