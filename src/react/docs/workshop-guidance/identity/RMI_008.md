#  Implementar Acesso Condicional para contas de convidado

**Esforço de Implementação:** Médio – Requer configuração de políticas e definições de acesso entre locatários.
**Impacto ao Usuário:** Alto – Os convidados podem precisar registrar MFA ou aceitar termos de uso.

## Visão Geral

Implementar o Acesso Condicional para contas de convidado envolve criar políticas que impõem requisitos de segurança para usuários externos que acessam os recursos da organização. Isso se alinha ao princípio de Zero Trust de *Verificação Explícita*, garantindo que cada solicitação de acesso de um usuário convidado seja autenticada e autorizada com base em condições definidas. Os principais controles incluem exigir autenticação multifator (MFA) para todos os usuários convidados, impor a aceitação de termos de uso e configurar controles de sessão para limitar a duração do acesso. É importante observar que determinadas políticas de Acesso Condicional, como as que exigem conformidade de dispositivo, requerem a configuração de definições de acesso entre locatários. Portanto, as organizações devem ajustar suas políticas para acomodar essas restrições enquanto mantêm os padrões de segurança. A falha em configurar adequadamente o Acesso Condicional para usuários convidados pode levar a acessos não autorizados e possíveis violações de dados.
