#  Migrar sistemas de MFA locais

**Esforço de Implementação:** Alto – A transição de sistemas de MFA locais para métodos nativos do Entra ID envolve desativação de infraestrutura, atualizações de políticas e fluxos de trabalho de novo registro de usuários.

**Impacto ao Usuário:** Médio – Os usuários devem registrar novos métodos e se adaptar a experiências de login atualizadas, embora a interrupção geralmente seja limitada à fase de registro.

## Visão Geral

Migrar de sistemas de MFA locais para a autenticação multifator do Microsoft Entra permite que as organizações centralizem e modernizem sua infraestrutura de autenticação. Essa migração é fundamental para aplicar controles de Zero Trust. Ela suporta o princípio de **Verificação Explícita** ao habilitar políticas de Acesso Condicional que avaliam sinais como saúde do dispositivo, risco do usuário e comportamento de login antes de conceder acesso. Também está ligada ao **Acesso com Menor Privilégio** ao permitir a aplicação adaptativa de requisitos de autenticação com base na sensibilidade do recurso ou no contexto de risco, garantindo que os usuários sejam desafiados somente quando necessário. Por fim, ao remover servidores de MFA locais da rede e consolidar a lógica de autenticação no Microsoft Entra ID, as organizações seguem o princípio de **Assumir Violação** ao reduzir a superfície de ataque, eliminar componentes não gerenciados e obter visibilidade unificada para detecção de ameaças. Não concluir essa migração deixa as organizações expostas a controles de segurança desatualizados e com fraca integração ao Acesso Condicional.
