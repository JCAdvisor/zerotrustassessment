#  Aplicar autenticação com credenciais fortes para todas as contas privilegiadas

**Esforço de Implementação:** Alto – Implementar métodos de autenticação forte para todas as contas privilegiadas requer esforços coordenados entre as equipes de TI, segurança e aplicativos para auditar, avaliar e reconfigurar mecanismos de autenticação, geralmente envolvendo múltiplos sistemas e partes interessadas.

**Impacto ao Usuário:** Baixo – Os administradores devem adotar novos métodos de autenticação, o que pode exigir treinamento e ajustes em seus fluxos de trabalho.

## Visão Geral

Aplicar a força de autenticação para todas as contas privilegiadas é essencial para proteger recursos sensíveis. Isso envolve exigir o uso de métodos de autenticação resistentes a phishing, como Windows Hello for Business, chaves de segurança FIDO2 ou autenticação baseada em certificado, por meio de políticas de Acesso Condicional no Microsoft Entra ID. Ao aproveitar o recurso de forças de autenticação, as organizações podem especificar os métodos de autenticação necessários para acessar recursos críticos, garantindo que apenas usuários verificados e autorizados possam realizar operações de alto privilégio. Essa abordagem está alinhada aos princípios de Zero Trust de **Verificação Explícita** ao validar continuamente as identidades dos usuários e **Acesso com Menor Privilégio** ao restringir o acesso com base em autenticação forte. A falha em implementar essas medidas pode expor as organizações a ataques baseados em credenciais, acesso não autorizado e potenciais violações de dados.
