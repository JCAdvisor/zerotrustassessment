#  Migrar listas de distribuição para Teams e Grupos do Microsoft 365

**Esforço de Implementação:** Alto – As equipes de TI devem identificar listas de distribuição elegíveis, atualizá-las via Centro de Administração do Exchange ou PowerShell e configurar as definições de Grupos do Microsoft 365 conforme necessário.

**Impacto ao Usuário:** Baixo – Os usuários continuarão usando endereços de e-mail familiares, mas ganharão novas funcionalidades como calendários compartilhados, arquivos e colaboração baseada no Teams.

## Visão Geral

Migrar listas de distribuição para Grupos do Microsoft 365 ajuda a modernizar a comunicação e fortalecer a governança de identidade. Ao contrário das listas de distribuição legadas, os Grupos do Microsoft 365 estão profundamente integrados ao Entra ID, habilitando o uso de Acesso Condicional, revisões de acesso e logs de auditoria. Isso suporta o princípio de Zero Trust de **Acesso com Menor Privilégio** ao permitir que as organizações revisem periodicamente a associação aos grupos e removam usuários que não precisam mais de acesso. Também está alinhado à **Verificação Explícita** porque o acesso é gerenciado por meio do Entra ID, onde a autenticação considera identidade do usuário, dispositivo e risco de login. A migração contribui ainda para o **Assumir Violação** ao permitir que as equipes de segurança rastreiem a atividade dos grupos e apliquem controles de proteção de dados e conformidade que não existem nas DLs clássicas.

Deixar as listas de distribuição não gerenciadas em ambientes Exchange ou locais significa perder a oportunidade de aplicar políticas de Zero Trust. Migrar para Grupos do Microsoft 365 fornece uma base para colaboração segura integrada ao ecossistema de segurança mais amplo do Microsoft 365.
