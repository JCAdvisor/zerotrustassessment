#  Implementar monitoramento do provisionamento de identidade da força de trabalho

**Esforço de Implementação:** Médio – Requer configurar definições de diagnóstico, integrar com o Azure Monitor e estabelecer pastas de trabalho e alertas para acompanhar a integridade do provisionamento.

**Impacto ao Usuário:** Baixo – O monitoramento é gerenciado pelos administradores e não requer nenhuma ação ou comunicação aos usuários finais.

## Visão Geral

Implementar monitoramento para o provisionamento de identidade da força de trabalho habilita supervisão contínua dos sistemas responsáveis pela sincronização de dados de usuários de sistemas de RH ou diretórios para o Microsoft Entra ID. Isso inclui habilitar logs de auditoria e diagnósticos, transmitir logs de provisionamento para o Azure Monitor e usar pastas de trabalho do Microsoft Entra, como Provisioning Insights, para detectar anomalias e validar a integridade do fluxo. Essa atividade reforça o princípio de Zero Trust de **Assumir Violação** ao garantir que os eventos de provisionamento de identidade sejam observáveis, verificáveis e sujeitos a detecção de anomalias. Também suporta a **Verificação Explícita** ao habilitar a validação de que as decisões de acesso são baseadas em dados oportunos e corretamente provisionados. Sem monitoramento, configurações incorretas ou eventos de provisionamento com falha podem passar despercebidos, levando a contas dormentes, acesso inadequado ou falhas de integração, todos os quais introduzem riscos operacionais e de segurança.
