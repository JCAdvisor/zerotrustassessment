#  Parar de criar contas locais para novos usuários externos

## Visão Geral

Use a colaboração B2B do Entra ID para novas Identidades externas (parceiros de negócios e convidados). As contas B2B oferecem vários benefícios em relação às contas de convidado locais para gerenciar colaborações externas:

* **Single Sign-On (SSO) e melhor experiência do usuário para colaboradores:** As contas B2B permitem que parceiros externos que também possuem Entra ID usem suas credenciais existentes para acessar recursos. Isso é mais conveniente para os usuários de outras organizações, pois não precisam lembrar informações de login adicionais e podem utilizar o SSO para uma experiência fluida. O B2B também permite a colaboração com organizações externas que não possuem Entra ID.

* **Redução da sobrecarga de gerenciamento de credenciais de convidados:** Ao usar contas B2B, sua organização não precisa gerenciar credenciais de usuários convidados. Isso reduz significativamente a sobrecarga administrativa de TI associada ao provisionamento e redefinição de credenciais.

* **Ferramentas nativas de nuvem para governar o ciclo de vida:** O Entra ID oferece recursos de governança de identidade nativos em nuvem que você pode implantar e configurar para controlar o ciclo de vida das contas de convidados. Exemplos incluem revisões de acesso e pacotes de acesso.

* **Simplificação dos controles de segurança:** As configurações de acesso entre tenants permitem configurar se deve-se confiar nas declarações de MFA ou dispositivo de organizações externas que também estejam no Entra ID. No contexto de Zero Trust, a declaração de saúde do dispositivo de organizações externas é uma consideração importante.
