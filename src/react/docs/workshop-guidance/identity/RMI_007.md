#  Implantar políticas de Acesso Condicional com controle de risco

**Esforço de Implementação:** Médio – Requer configuração de políticas baseadas em risco e integração com o Identity Protection.
**Impacto ao Usuário:** Médio – Os usuários podem observar prompts de autenticação adicionais, redefinições de senha ou bloqueio de acesso.

## Visão Geral

Implantar políticas de Acesso Condicional que visam usuários arriscados e entradas arriscadas aproveita as avaliações de risco em tempo real do Microsoft Entra ID Protection para impor controles de acesso adaptativos. As políticas de risco de entrada avaliam a probabilidade de uma tentativa de entrada ser não autorizada, solicitando ações como autenticação multifator (MFA) para entradas de risco médio ou alto. As políticas de risco de usuário avaliam a probabilidade de as credenciais de um usuário estarem comprometidas, exigindo alterações seguras de senha para usuários de alto risco. Essas medidas incorporam o princípio de Zero Trust de *Verificação Explícita*, analisando continuamente o comportamento do usuário e os padrões de entrada para detectar anomalias. Sem essas políticas, as organizações correm o risco de acesso não autorizado a partir de contas comprometidas, pois agentes de ameaça poderiam explorar a ausência de controles dinâmicos baseados em risco para se infiltrar nos sistemas sem ser detectados.
