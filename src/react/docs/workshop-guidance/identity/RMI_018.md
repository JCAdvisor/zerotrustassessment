#  Migrar SSO para aplicativos baseados em Web Access Management de colaboradores

**Esforço de Implementação:** Alto – As equipes de TI dos clientes precisam executar projetos de migração envolvendo múltiplos proprietários de aplicativos e stakeholders técnicos.

**Impacto ao Usuário:** Alto – Os colaboradores devem ser informados e possivelmente retreinados devido a mudanças na experiência do usuário após a migração.

## Visão Geral

Migrar o Single Sign-On (SSO) para aplicativos baseados em Web Access Management (WAM) envolve transferir a autenticação de soluções legadas locais, como Oracle Access Manager ou CA SiteMinder, para o Microsoft Entra ID. Essa migração centraliza o gerenciamento de identidade do usuário e melhora a segurança ao aproveitar as políticas de Acesso Condicional do Entra. Ela está estreitamente alinhada ao princípio de Zero Trust de **Verificação Explícita**, pois a autenticação no Entra ID utiliza verificação contínua de identidade do usuário, saúde do dispositivo e dados de localização. Também suporta o **Acesso com Menor Privilégio**, habilitando controles de política adaptativa baseados em risco. Não concluir essa migração mantém a dependência de sistemas desatualizados que carecem de recursos de segurança modernos, aumentando a vulnerabilidade a ataques baseados em identidade por agentes de ameaças, pois sistemas legados raramente se adaptam a riscos e violações em tempo real.

O Web Access Management (WAM) é um sistema especializado de gerenciamento de identidade projetado para controlar e proteger o acesso a recursos e aplicativos baseados na web.
