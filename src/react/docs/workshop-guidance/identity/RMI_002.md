#  Pare de comprar ou desenvolver aplicativos dependentes do Active Directory

**Esforço de Implementação:** Alto – A transição para longe das dependências do Active Directory (AD) requer mudanças abrangentes em políticas de aquisição, práticas de desenvolvimento e processos de governança de TI.&#x20;

**Impacto ao Usuário:** Baixo – Essas mudanças são principalmente internas às equipes de TI e desenvolvimento, com impacto direto mínimo aos usuários finais.

## Visão Geral

Cessar a aquisição ou o desenvolvimento de aplicativos que dependem do Active Directory (AD) é uma decisão estratégica em direção à adoção de um modelo de segurança cloud-first com Zero Trust. Aplicativos dependentes do AD frequentemente requerem infraestrutura local, dificultando a escalabilidade e a implementação de controles de segurança modernos. Ao evitar novas dependências do AD, as organizações podem aproveitar os recursos do Microsoft Entra ID, como Acesso Condicional e autenticação multifator, para aplicar políticas baseadas em identidade do usuário, integridade do dispositivo e outros fatores contextuais.&#x20;

Essa abordagem se alinha aos princípios de Zero Trust, garantindo a verificação explícita das solicitações de acesso e minimizando a confiança implícita baseada na localização de rede. Ela também suporta o princípio de menor privilégio, habilitando controles de acesso granulares e reduzindo a superfície de ataque.&#x20;

Centralizar a autenticação de aplicativos no Entra ID também traz benefícios adicionais bem recebidos por TI, SecOps e/ou usuários finais, como:
- Logs de entrada centralizados;
- Consistência de login e Single Sign-On (SSO);
- Mecanismo central de políticas de acesso (Acesso Condicional) para cada aplicativo/serviço (local, nuvem ou multinuvem);
- Funcionalidades de Governança de Identidade, como Revisões de Acesso, Gerenciamento de Direitos, fluxos de trabalho de ciclo de vida (fluxos automatizados para integração, desligamento e alterações de atributos), entre outros;
- Recursos nativos do Entra ID como Redefinição de Senha de Autoatendimento (SSPR), Controle de Acesso Baseado em Função (RBAC), conformidade regulatória, disponibilidade global, entre outros;
- Recursos do Global Secure Access, como Private Access (ZTNA), Secure Web Gateway, entre outros.

A não adoção de aplicativos independentes do AD pode resultar em maior complexidade, custos de manutenção elevados e exposição a vulnerabilidades de segurança devido a mecanismos de autenticação desatualizados. Adotar aplicativos nativos de nuvem integrados ao Microsoft Entra ID melhora a postura de segurança e a eficiência operacional.
