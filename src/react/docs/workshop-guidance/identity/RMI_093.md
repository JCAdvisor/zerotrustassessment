#  Implantar conectores de provisionamento para fontes de dados de identidade

**Esforço de Implementação:** Alto – Implantar conectores de provisionamento requer configuração de agentes de provisionamento do Microsoft Entra, integração com sistemas como Active Directory ou LDAP e coordenação com sistemas de RH ou TI, o que constitui um projeto significativo para as equipes de TI e SecOps.

**Impacto ao Usuário:** Baixo – A atividade é realizada pelos administradores e não requer nenhuma ação ou notificação para os usuários finais.

## Visão Geral

Implantar conectores de provisionamento para fontes de dados de identidade habilita a sincronização automatizada de atributos de identidade da força de trabalho de sistemas como Active Directory, diretórios LDAP e plataformas de RH para o Microsoft Entra ID. Essa automação é essencial para manter um registro de identidade consistente e autoritativo em sistemas em nuvem e locais. Suporta o princípio de Zero Trust de **Verificação Explícita** ao garantir que as políticas e controles de acesso operem com dados de identidade precisos e oportunos, e o **Acesso com Menor Privilégio** ao alinhar os direitos a atributos baseados em funções de fontes confiáveis. Sem conectores de provisionamento, erros manuais, dados obsoletos ou atualizações tardias podem levar a superprovisionamento, contas órfãs ou falhas de conformidade, que agentes de ameaças podem explorar para obter acesso não autorizado persistente.
