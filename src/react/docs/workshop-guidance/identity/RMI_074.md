#  Integrar todos os logs do Entra ao SIEM

**Esforço de Implementação:** Médio – Configurar a integração abrangente de logs requer configurar definições de diagnóstico, gerenciar pipelines de dados para o SIEM e garantir políticas adequadas de armazenamento e retenção.

**Impacto ao Usuário:** Baixo – Este é um processo administrativo de backend sem impacto direto na experiência do usuário final.

## Visão Geral

Integrar os logs do Microsoft Entra ID a um sistema de Gerenciamento de Informações e Eventos de Segurança (SIEM) é crítico para monitoramento centralizado, detecção de ameaças e resposta a incidentes. No mínimo, exporte os seguintes logs:

- AuditLogs
- SignInLogs
- ServicePrincipalSignInLogs
- ManagedIdentitySignInLogs
- ADFSSignInLogs
- RiskyUsers
- UserRiskEvents
- RiskyServicePrincipals
- ServicePrincipalRiskEvents

Considere exportar outros tipos de log também para auxiliar investigações. Algumas dessas fontes de log podem gerar um grande volume de logs, portanto, certifique-se de ter uma estratégia de retenção de logs para controlar os custos de armazenamento:

- NonInteractiveUserSignInLogs
- ProvisioningLogs
- NetworkAccessTrafficLogs
- EnrichedOffice365AuditLogs
- MicrosoftGraphActivityLogs
- RemoteNetworkHealthLogs

Essa integração está alinhada aos princípios de Zero Trust ao habilitar a verificação contínua de atividades de usuários e dispositivos (verificação explícita), garantindo que as decisões de acesso sejam baseadas em dados abrangentes e em tempo real. Suporta o princípio de assumir violação ao fornecer visibilidade sobre potenciais incidentes de segurança, permitindo detecção e resposta rápidas a ameaças. Sem essa integração, as organizações correm o risco de detecção tardia de atividades maliciosas, capacidades forenses limitadas e não conformidade com requisitos regulatórios.
