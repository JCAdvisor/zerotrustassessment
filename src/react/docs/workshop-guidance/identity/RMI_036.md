#  Ativar a Sincronização de Hash de Senha

**Esforço de Implementação:** Médio – Habilitar a Sincronização de Hash de Senha (PHS) requer a configuração do Microsoft Entra Connect e a garantia de permissões e conectividade adequadas entre o Active Directory local e o Microsoft Entra ID.

**Impacto ao Usuário:** Baixo – Os usuários continuam usando suas credenciais existentes sem alterações na experiência de login.

## Visão Geral

Habilitar a Sincronização de Hash de Senha (PHS) em um ambiente de identidade híbrida envolve configurar o Microsoft Entra Connect para sincronizar hashes de senha do Active Directory local para o Microsoft Entra ID. Essa configuração melhora a segurança e suporta os princípios de Zero Trust ao fornecer um método de autenticação resiliente e habilitar recursos avançados de detecção de ameaças.

Ao sincronizar hashes de senha, as organizações podem detectar credenciais vazadas e ataques de pulverização de senhas por meio do Microsoft Entra ID Protection. Isso está alinhado ao princípio de **Verificação Explícita** ao avaliar continuamente o risco do usuário com base na exposição de credenciais. Além disso, o PHS serve como método de autenticação de backup em ambientes federados, garantindo continuidade de acesso durante interrupções, o que suporta o princípio de **Assumir Violação** ao manter a postura de segurança mesmo quando os métodos de autenticação primários falham.

Negligenciar a habilitação do PHS pode resultar em falhas na detecção de credenciais comprometidas e maior dependência de métodos de autenticação potencialmente vulneráveis. Implementar o PHS é um passo estratégico para fortalecer o framework de segurança de identidade da organização.
