#  Definir Esquema de Atributos, Semântica e Fluxos de Dados

**Esforço de Implementação:** Alto – Requer colaboração multifuncional para definir esquemas de atributos, avaliar a qualidade dos dados e implementar extensões de esquema no Microsoft Entra ID e no Active Directory.

**Impacto ao Usuário:** Baixo – O processo é gerenciado pelos administradores e não requer ação direta ou notificação para os usuários finais.

## Visão Geral

Definir o esquema de atributos, a semântica e os fluxos de dados é uma etapa crítica no estabelecimento de um sistema robusto de gerenciamento de identidade. Isso envolve identificar os atributos necessários pelos aplicativos, determinar suas fontes (como sistemas de RH ou Active Directory) e mapeá-los com precisão para o Microsoft Entra ID. Desafios frequentemente surgem quando os atributos necessários estão ausentes ou são de baixa qualidade nos sistemas de origem, necessitando limpeza de dados ou extensões de esquema no Entra ID e no Active Directory. Estabelecer correlação confiável (ancoragem) entre as fontes de dados e o Entra ID é essencial para evitar contas duplicadas e garantir provisionamento preciso. Esse processo está alinhado aos princípios de Zero Trust de **Verificação Explícita** ao garantir que as decisões de autenticação sejam baseadas em dados precisos e abrangentes, e de **Acesso com Menor Privilégio** ao habilitar controle de acesso preciso com base em atributos bem definidos. Negligenciar essa etapa pode levar a erros de provisionamento, vulnerabilidades de segurança e problemas de conformidade.
