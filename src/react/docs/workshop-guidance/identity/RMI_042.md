#  Promover o uso de métodos de autenticação sem senha

**Esforço de Implementação:** Médio – As equipes de TI precisam habilitar métodos sem senha no Microsoft Entra ID, configurar o Acesso Condicional para exigir seu uso, monitorar a atividade de registro e login, e tomar medidas para impulsionar a adoção em toda a organização.

**Impacto ao Usuário:** Médio – Os usuários devem registrar um método sem senha, como Windows Hello for Business, Microsoft Authenticator ou uma chave de segurança FIDO2, e aprender a usá-lo para login. A maioria dos usuários já estará familiarizada com ferramentas semelhantes do MFA.

## Visão Geral

Incentivar os usuários a registrar e usar métodos de login sem senha, como Microsoft Authenticator ou chaves de segurança FIDO2, melhora tanto a segurança quanto a experiência de login. Esses métodos são resistentes a phishing porque vinculam o login a um dispositivo específico e exigem algo que o usuário possui, como impressão digital ou PIN. Isso suporta o princípio de Zero Trust de **Verificação Explícita** ao verificar múltiplos sinais fortes no login, e o princípio de **Assumir Violação** ao dificultar o sucesso de agentes de ameaças mesmo que possuam um nome de usuário.

Promover o uso vai além de apenas ativar o recurso. As equipes de TI devem rastrear quantos usuários registraram métodos sem senha, reportar o progresso à liderança e definir políticas de Acesso Condicional que exijam esses métodos — especialmente para aplicativos ou funções sensíveis. Sem esse esforço, os usuários podem voltar a usar senhas, que continuam sendo o principal alvo dos atacantes.
