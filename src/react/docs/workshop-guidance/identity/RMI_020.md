#  Migrar SSO para aplicativos federados de Identidades Externas

**Esforço de Implementação:** Alto – Requer reconfiguração de fluxos de autenticação e coordenação com parceiros externos para realizar a transição da confiança federada.

**Impacto ao Usuário:** Alto – Os usuários externos passarão por mudanças na forma como se autenticam, exigindo comunicação e gerenciamento de mudanças.

## Visão Geral

Esta atividade envolve migrar aplicativos federados usados por identidades externas (como parceiros ou contratados) de provedores de identidade legados para o Microsoft Entra External ID. Esses aplicativos podem atualmente depender de configurações de federação personalizadas ou desatualizadas que não suportam a aplicação de políticas modernas. Migrá-los para o Entra External ID permite que as organizações apliquem Acesso Condicional, controles de governança e melhorem a telemetria de autenticação além dos limites do tenant.

Isso está diretamente alinhado à **Verificação Explícita**, pois o Entra External ID habilita a avaliação de acesso baseada em políticas para usuários externos usando condições sensíveis ao contexto, como risco do usuário, e controles mais robustos como autenticação forte. A mudança também habilita o **Assumir Violação** ao substituir a confiança implícita baseada em identidade federada por avaliação contínua de acesso e restrições entre tenants. Os aplicativos movidos para o Entra External ID ganham visibilidade por meio de logs unificados e detecção aprimorada de ameaças.

Se essa atividade for ignorada, as identidades externas continuarão se autenticando por caminhos de confiança não gerenciados ou fracos, contornando a aplicação central de Zero Trust e expondo a organização a ameaças decorrentes de protocolos de federação desatualizados, logs insuficientes e controle limitado sobre o comportamento de acesso externo.
