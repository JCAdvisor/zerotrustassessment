#  Descobrir funções privilegiadas existentes

**Esforço de Implementação:** Médio – As equipes de TI e segurança devem identificar os detentores de funções privilegiadas, analisar seu acesso e determinar se alternativas com menor privilégio são adequadas.

**Impacto ao Usuário:** Baixo – A maioria dos usuários não perceberá uma mudança, a menos que seu nível de acesso seja reduzido após a revisão; comunicação e suporte serão necessários para quaisquer mudanças.

## Visão Geral

Descobrir quem detém funções privilegiadas na organização é um passo fundamental para aplicar o menor privilégio e os princípios de Zero Trust. Esse processo começa com a criação de um inventário de todos os usuários com funções elevadas, usando ferramentas como o Microsoft Entra ID ou o Microsoft Graph. Uma vez que essa lista esteja completa, cada atribuição deve ser revisada para entender quais permissões são realmente usadas e se correspondem às responsabilidades do cargo do usuário. Se os usuários tiverem acesso mais amplo do que o necessário, a função deve ser ajustada para uma mais adequada. Esse trabalho suporta o **Acesso com Menor Privilégio** ao reduzir ativamente permissões desnecessárias, a **Verificação Explícita** ao basear as decisões de atribuição de funções em evidências de trabalho e padrões de uso documentados, e o **Assumir Violação** ao reduzir o raio de impacto de uma conta privilegiada comprometida. Sem um inventário completo e revisão regular do acesso privilegiado, as organizações se expõem ao superprovisionamento, ameaças internas e pontos cegos na governança de identidade.
