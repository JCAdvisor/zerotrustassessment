#  Implementar métodos sem senha do Authenticator

**Esforço de Implementação:** Baixo – Como o aplicativo Authenticator já está implantado para MFA e SSPR, apenas etapas mínimas de configuração e habilitação são necessárias para permitir o login sem senha.

**Impacto ao Usuário:** Baixo – Os usuários já têm o aplicativo instalado; eles só precisam concluir uma etapa rápida de registro para habilitar o login sem senha.

## Visão Geral

Implementar a autenticação sem senha com o Microsoft Authenticator permite que os usuários façam login sem inserir uma senha, aprovando uma notificação no dispositivo móvel registrado. Esse método fortalece a segurança e simplifica a experiência do usuário, alinhando-se aos princípios fundamentais de Zero Trust.

Se os usuários já utilizam o Microsoft Authenticator para MFA e SSPR, habilitar o login sem senha é uma melhoria leve mas de alto impacto. Ela suporta diretamente o princípio de **Verificação Explícita** ao vincular o login a um dispositivo específico e registrado, exigindo verificação biométrica ou por PIN nesse dispositivo. Também ajuda a aplicar o **Assumir Violação** ao remover senhas do fluxo de login, o que reduz a exposição a ataques de phishing, replay e pulverização de senhas. Esse método aproveita o framework de força de autenticação do Microsoft Entra ID e se integra perfeitamente às políticas de Acesso Condicional.

Ignorar essa implantação significa continuar dependendo de fluxos baseados em senha que permanecem vulneráveis a roubo e comprometimento. Habilitar o login sem senha pelo telefone completa a transição para um login moderno e resistente a phishing para usuários que já possuem as ferramentas certas disponíveis.
